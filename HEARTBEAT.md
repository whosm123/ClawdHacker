# HEARTBEAT.md - 渗透测试监控检查

## 每日检查项

### 1. 工具状态检查
- [ ] Kali工具更新检查 (`apt update && apt list --upgradable`)
- [ ] 关键工具版本验证 (nmap, sqlmap, nuclei等)
- [ ] 自定义脚本功能测试

### 2. 测试环境检查
- [ ] 网络连接状态 (目标可达性)
- [ ] 代理服务状态 (Burp/ZAP)
- [ ] 工作目录空间检查
- [ ] 日志文件大小监控

### 3. 安全监控
- [ ] 异常登录尝试检查 (`/var/log/auth.log`)
- [ ] 系统资源使用监控 (CPU/内存/磁盘)
- [ ] 网络流量异常检测
- [ ] 防火墙规则验证

### 4. 测试进度检查
- [ ] 当前测试目标状态
- [ ] 已发现漏洞统计
- [ ] 报告生成进度
- [ ] 修复验证状态

## 每周检查项

### 1. 知识库更新
- [ ] 新漏洞情报收集 (CVE/NVD)
- [ ] 工具新技术学习
- [ ] 测试方法论更新

### 2. 环境维护
- [ ] 系统安全补丁应用
- [ ] 工具链优化
- [ ] 备份完整性验证

### 3. 流程改进
- [ ] 测试效率分析
- [ ] 误报率统计
- [ ] 客户反馈整理

## 应急检查项

当检测到以下情况时立即通知：
- 目标服务不可达超过30分钟
- 发现零日漏洞或高危漏洞
- 测试环境安全事件
- 客户紧急联系请求

## 检查频率

- **高频检查** (每2小时): 工具状态、网络连接
- **日常检查** (每日): 测试进度、安全监控
- **周期检查** (每周): 知识更新、环境维护

## 状态跟踪

在 `memory/heartbeat-state.json` 中记录检查状态：

## 响应策略

### 正常情况
- 所有检查通过 → `HEARTBEAT_OK`

### 警告情况
- 工具需要更新 → 记录，计划更新
- 测试进度延迟 → 调整计划
- 资源使用偏高 → 优化配置

### 紧急情况
- 安全事件 → 立即停止测试，通知客户
- 服务中断 → 切换备用方案
- 高危漏洞 → 优先处理，紧急报告

_保持监控，及时响应，确保测试安全有效进行。_
