## 渗透测试方法论

### 1. 结构化工作流程
```
1. 目标确认与授权验证
2. 信息收集与侦察  
3. 漏洞扫描与发现
4. 深入验证与证据收集
5. 漏洞报告生成
6. 修复验证与复测
```

### 2. 输出标准
- **格式**: 强制JSON schema
- **验证**: 所有结论必须工具验证
- **证据**: 可复现的证据链
- **风险**: 最小化、低风险测试步骤

### 3. 风险控制原则
- 🚫 禁止破坏性payload
- 🚫 禁止横向移动  
- 🚫 禁止越权访问敏感数据
- ✅ 仅授权范围内测试
- ✅ 非破坏性验证优先

## 工具使用经验

### Kali工具最佳实践
1. **nmap**: 使用`--script "not intrusive"`进行安全扫描
2. **sqlmap**: 使用`--test-filter`避免破坏性操作
3. **nuclei**: 优先使用`-severity critical,high`过滤
4. **自定义脚本**: 保存在`/root/scripts/`，版本控制

### 证据收集规范
- **截图**: PNG格式，包含URL和时间戳
- **HTTP流量**: HAR格式或原始请求/响应
- **工具输出**: 原始输出 + 分析注释
- **时间线**: 按时间顺序记录测试活动

## 漏洞分类与处理

### 严重等级定义
- **Critical**: 远程代码执行、权限提升、数据泄露
- **High**: SQL注入、XSS、CSRF、SSRF
- **Medium**: 信息泄露、不安全的直接对象引用
- **Low**: 安全头缺失、版本信息泄露
- **Info**: 配置信息、技术栈识别

## 重要决策与教训

### 安全测试边界
- 始终在授权范围内测试
